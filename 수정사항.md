# 🚀 업데이트 내역 (2026.05.13)

## 1. 생성형 AI (LLM) 버전 업그레이드
- **대상 파일**: `app/llm/llm_client.py`, `app/config.py`
- **변경 사항**: 쿼리 확장(Query Expansion) 및 의도 분석에 사용되는 Gemini LLM 모델을 기존 `gemini-2.0-flash`에서 최신 모델인 `gemini-3.1-flash-lite`로 상향 조정했습니다. 이를 통해 응답 속도 및 확장 키워드 품질을 최적화했습니다.

## 2. 임베딩 모델(Dense Retrieval) 아키텍처 버그 수정
- **대상 파일**: `app/pipeline/stage1_ranking_pipeline.py`
- **변경 사항**: 의존성 주입(Dependency Injection) 오류로 인해 `DenseRetriever` 내부의 훌륭한 투트랙 임베딩 기능이 무력화되던 버그(Bypass Bug)를 해결했습니다.
- **효과**: 이제 사용자의 검색어(Goal)는 `RETRIEVAL_QUERY`로, 일상 로그(Log)는 `RETRIEVAL_DOCUMENT`로 명확히 나뉘어 임베딩되며, **비대칭적 의미 검색(Asymmetric Semantic Search)**이 정상 작동하여 후보군 재현율(Candidate Recall)이 상승합니다.

## 3. 디버깅 편의성 및 가시성 개선
- **대상 파일**: `app/retrieval/candidate_retrieval.py`
- **변경 사항**: 실제 1차 검색(Dense Retrieval) 시 Gemini 임베딩 API에 투입되는 텍스트(`dense_query`)를 터미널에 명시적으로 출력해주는 기능(`[ACTUAL DENSE QUERY FED TO EMBEDDING MODEL]`)을 추가했습니다. (임시 하드코딩 테스트 코드는 주석으로 보존)
- **대상 파일**: `app/retrieval/query_expansion.py`
- **변경 사항**: 쿼리 확장 로직의 영문 주석을 상세한 한글 설명으로 번역하고, 범용 단어 필터링(`_GENERIC_TERMS`) 리스트를 미세 조정했습니다.

## 4. Windows 터미널 인코딩(Emoji) 에러 해결
- **대상 파일**: `scripts/run_stage1.py`
- **변경 사항**: 윈도우 환경(CP949)에서 스크립트 실행 시 진단 결과의 이모지(⚠, ✓)로 인해 스크립트가 강제 종료되던 `UnicodeEncodeError`를 해결하기 위해, 해당 기호를 일반 텍스트(`[!]`, `[v]`)로 교체했습니다.
